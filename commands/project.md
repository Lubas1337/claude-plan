# /claude-plan:project — Проектный контекст

## Инструкции

> При вопросах пользователю — используй `AskUserQuestion` tool с опциями (селектор), а не текстовые вопросы.

Создаёт или обновляет `.plan/PROJECT.md` — проектный контекст, объединяющий все планы.

### Step -1: Detect Backend

Выполни процедуру **"Detect Backend"** из SKILL.md:
1. Проверь `.plan/config.json` → определи `mode` (local/obsidian) и `vaultPath`
2. Если `mode === "local"` → текущее поведение, PROJECT.md в `.plan/PROJECT.md`
3. Если `mode === "obsidian"`:
   - PROJECT.md → `<vault>/Project.md` (корень vault)
   - Формат: YAML frontmatter с `type: project`, wikilinks в Roadmap таблице
   - В Roadmap используй `[[Plan Name]]` вместо `[plan-name](plan-name/MASTER.md)`

### Шаг 0: Определи режим

Проверь существование `.plan/PROJECT.md`:
- **Есть** → режим "Обновить" (Шаг 2)
- **Нет** → режим "Создать" (Шаг 1)

### Шаг 1: Создание (интерактивное обсуждение)

#### 1a: Сбор контекста

- Прочитай `.plan/INDEX.md` — какие планы уже есть
- Для каждого плана прочитай MASTER.md: vision, goals
- Выполни процедуру "Project Context" из SKILL.md

#### 1b: Обсуждение — 3 раунда

**Раунд 1: Миссия и Vision**

Спроси через `AskUserQuestion` с текстовым вводом (Other):
"Опиши одним-двумя предложениями: зачем существует этот проект и куда он идёт?"

AI анализирует ответ + контекст существующих планов.
Предлагает структурированную формулировку Mission + Vision.

Спроси через `AskUserQuestion`:
- "Хорошо, продолжаем" → Раунд 2
- "Хочу уточнить" (Other) → прими уточнение, предложи заново

**Раунд 2: Стратегические цели**

На основе Mission/Vision + существующих планов, AI предлагает 3-7 стратегических целей.

Покажи предложенные цели и спроси через `AskUserQuestion`:
- "Цели хорошие" → Раунд 3
- "Хочу изменить" (Other) → прими изменения

**Раунд 3: Roadmap и Constraints**

AI группирует существующие планы в фазы roadmap.
Предлагает пустые фазы для будущих планов.
Предлагает constraints на основе контекста.

Спроси через `AskUserQuestion`:
- "Создать PROJECT.md" → Шаг 3
- "Хочу изменить" (Other) → прими правки

### Шаг 2: Обновление

Прочитай текущий `.plan/PROJECT.md`.

Спроси через `AskUserQuestion` с опциями:
- "Обновить Mission/Vision" → спроси новую формулировку
- "Добавить/изменить цели" → покажи текущие, спроси что менять
- "Обновить Roadmap" → синхронизируй с текущими планами из INDEX.md
- "Добавить решение" → добавить запись в Key Decisions
- "Полная ревизия" → пройди все раунды как при создании
- "Синхронизировать" → авто-обновить Roadmap statuses из INDEX.md

### Шаг 3: Запись файла

Создай/обнови `.plan/PROJECT.md` по формату из SKILL.md.

### Шаг 4: Вывод

Покажи созданный/обновлённый PROJECT.md.
Подсказка: "PROJECT.md будет учитываться при создании новых планов (/claude-plan:init)"

### Правила

- PROJECT.md — опциональный. Проекты без него работают как раньше.
- Не создавай PROJECT.md автоматически. Только по явному вызову команды.
- При обновлении Roadmap: если план completed → фаза done когда все планы фазы completed.
- Максимум 7 стратегических целей.
- Key Decisions: дата + описание. Ссылка на ADR если есть.
