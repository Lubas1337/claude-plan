# /claude-plan:init — Создание нового плана

## Инструкции

Ты создаёшь новый план в `.plan/`. Skill "plan" содержит форматы файлов и конвенции. Используй его как справку.

### Шаг 1: Собери контекст и информацию

#### 1a. Загрузи проектный контекст

Выполни процедуру **"Project Context"** из SKILL.md:
- Из CLAUDE.md (уже в контексте): извлеки Stack, Project Structure, Architecture, Anti-Patterns
- Прочитай project memory (`~/.claude/projects/<path-hash>/memory/MEMORY.md`) — прошлые решения
- `ls .claude/skills/` — доступные навыки для аннотаций задач
- `ls internal/` (или исходную директорию) — реальные пакеты

Используй собранный контекст при разбиении на части, заполнении Files Affected и создании задач.

#### 1b. Спроси у пользователя

Спроси у пользователя (если не указано в аргументах):
1. **Название плана** (slug для директории, например `ai-features`, `lms-integration`)
2. **Vision** — одно предложение, что план достигает
3. **Goals** — 2-5 целей
4. **Части** — список частей с названиями, зависимостями и размерами (S/M/L/XL)

Если пользователь даёт описание в свободной форме — сам разбей на части, учитывая архитектурные слои проекта.

### Шаг 2: Создай структуру

```
.plan/
├── INDEX.md                          # Создай или обнови
└── <plan-name>/
    ├── MASTER.md
    ├── STATUS.md
    ├── CHANGELOG.md
    ├── research/
    │   └── adr/
    └── parts/
        ├── 01-<slug>/
        │   └── META.md
        ├── 02-<slug>/
        │   └── META.md
        └── ...
```

### Шаг 3: Заполни файлы

1. **MASTER.md** — по формату из SKILL.md. Таблица частей с зависимостями.
2. **META.md** для каждой части — статус `ready` если нет зависимостей, `blocked` если есть незавершённые.
3. **STATUS.md** — начальный дашборд (все blocked/ready).
4. **CHANGELOG.md** — пустой шаблон с `# Changelog`.
5. **INDEX.md** — добавь новый план в таблицу (создай файл если не существует).

### Шаг 4: Задачи для частей

Для каждой части создай task files в `parts/NN-slug/`:
- Разбей часть на 2-5 конкретных задач
- Каждая задача — отдельный файл `01-task.md`, `02-task.md`
- Формат task file — из SKILL.md

#### Agent Assignment

Для каждой создаваемой части предложи значение `Agent:`:
- **Size S** → `Agent: none` (дешевле выполнить inline)
- **Size M с Delegatable: yes** → `Agent: auto`
- **Size L/XL** → `Agent: auto` (рекомендуй делегирование для больших частей)
- Если часть research-focused → `Agent: Explore`
- Если часть содержит архитектурные решения → `Agent: architect`

Добавь колонку Agent в таблицу частей MASTER.md и в META.md каждой части.

Спроси пользователя: "Назначить агентов? Можно изменить позже в META.md"

### Правила

- Slug плана: lowercase, дефисы, без пробелов (`ai-features`, не `AI Features`)
- Номера частей: двузначные с нулём (`01`, `02`, ..., `10`)
- Первая часть без зависимостей — `ready`, остальные — `blocked` или `ready` по графу
- Не создавай task files если пользователь не описал задачи детально — оставь только META.md
- Добавляй `Agent:` в META.md только если пользователь подтвердил назначение агентов
- Размер (S/M/L/XL): S = 1-2 задачи, M = 3-5, L = 6-10, XL = 10+
- Files Affected в META.md — используй реальные пути из структуры проекта (не выдумывай)
- Задачи по конвенциям: если часть создаёт новый пакет — добавь задачу на `doc.go`
- При наличии релевантных skills — добавь `Skill hint: <name>` в Notes секции META.md
- Учитывай Anti-Patterns из CLAUDE.md при формулировке задач

### Вывод

После создания покажи:
1. Дерево созданных файлов
2. Таблицу частей из MASTER.md
3. Подсказку: `/claude-plan:next <plan-name>` для начала работы
